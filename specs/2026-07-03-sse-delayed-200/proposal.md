# Proposal: SSE 通路延迟 200 响应

在 SSE 通路，不要立即返回 200，等待一个 heartbeat interval，如果这时候上游已经返回 http response 了，则直接透传，没有返回，才由我们返回 200。

## 背景参考

本需求是对 [specs/2026-07-03-llm-sse-gateway](../2026-07-03-llm-sse-gateway/proposal.md) 中 SSE 包装通路"先行 200"行为的修改，原 spec 保持不变，仅作参考。

## 澄清补充（2026-07-03）

- **透传后的心跳**：等待窗口内上游返回 2xx 流式响应并透传后，流中途的长停顿仍要注入心跳保活——透传上游的状态码和响应头（解决原设计中 `x-request-id` 等上游头丢失的问题），但 body 仍走现有的解压 → 桥接 → 重编码通路，空闲时发 `":\n\n"`。
- **等待窗口内的上游结果**：只要上游返回了 HTTP 响应（含非 2xx，如 429/500），就完整透传给客户端；TCP 层出错（连接失败、连接超时等）则由网关返回 502。
